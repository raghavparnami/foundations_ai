# Loom v0.1 ‚Äî 6-hour demo build

**Deadline**: Monday 2026-05-18 (presentation). Built Saturday 2026-05-16.
**Goal**: A 3‚Äì5 minute demo that shows Loom's defining property ‚Äî "always
preparing" ‚Äî against a realistic manufacturing dataset.

## Scope cuts vs CLAUDE.md spec

These are out-of-scope for the demo (deferred to post-presentation):

- **Monorepo / pnpm workspaces** ‚Äî replaced with a single Next.js app with
  `src/lib/{catalog,agent,worker,mcp}` folders. Refactor to workspaces later.
- **graphile-worker** ‚Äî replaced with in-process workers triggered on app
  boot and via API routes (`/api/profile`, `/api/enrich`). For a 6h demo
  the queue infra is dead weight.
- **Plan mode, write tools, skills system, audit log UI** ‚Äî deferred to v0.3.
  Demo runs in execute mode with read tools only, which is the v0.1 spec anyway.
- **Lineage / Loop 3** ‚Äî deferred.
- **Auth, multi-tenancy, Clerk** ‚Äî deferred (v0.5).
- **OpenAI embeddings** ‚Äî optional. With ~6 demo tables, the entire catalog
  fits in the system prompt; pgvector schema is still built but only populated
  if `OPENAI_API_KEY` is present. Demo path doesn't depend on it.
- **Vitest + testcontainers** ‚Äî no tests for the demo. Manual verification only.
- **Drizzle migrations** ‚Äî `db push` only; no migration files.

## What ships in 6h

1. **Local Postgres in Docker** with two databases:
   - `loom_catalog` ‚Äî Loom's metadata store (pgvector enabled)
   - `loom_demo_source` ‚Äî manufacturing dataset the demo queries against
2. **Manufacturing seed data** (~5 tables, realistic noise):
   - `production_runs`, `deviations`, `equipment`, `operators`, `quality_checks`
3. **Structural crawl worker** (Loop 1, blocking the agent on first run):
   - Reads `information_schema` of the source DB
   - Profiles each column (null rate, distinct count, min/max, top-N, simple histogram)
   - Writes structural markdown to `loom-catalog/` on disk + rows in `catalog.tables`/`catalog.columns`
4. **Semantic enrichment worker** (Loop 2, manual trigger button + auto-runs after Loop 1):
   - Calls Claude Sonnet 4.6 with structural doc + 5 sample rows + column profile
   - Generates semantic doc (what the table is about, common joins, column meanings)
   - Stores generated paragraphs with `<!-- provenance: claude -->` tags
5. **Chat UI** (Next.js 15 + Vercel AI SDK + Claude Agent SDK):
   - Streaming chat with `useChat`
   - Agent has 4 read tools: `list_tables`, `describe_table`, `sample_rows`, `run_sql` (SELECT-only via `node-sql-parser`)
   - System prompt includes the full catalog index (table names + one-line summaries)
   - When a table is named, full markdown for that table is injected
6. **Live "preparing" panel**:
   - Right-side panel shows tables being profiled in real time
   - Each table card shows: profiling ‚Üí enriching ‚Üí ready
   - Click a table ‚Üí see the generated markdown with provenance tags color-coded
7. **Demo dataset polish**:
   - Pre-canned questions in the chat input as suggestions
   - One "wow" question: "What's our deviation rate by production line over
     the last 30 days, and which equipment is most often involved?"

## File-level diff

All paths relative to `~/loom/`.

### Foundations (Hour 0‚Äì1)

- `package.json` ‚Äî single-app, dependencies pinned
- `tsconfig.json` ‚Äî strict, `noUncheckedIndexedAccess: true`
- `next.config.ts` ‚Äî minimal

- `.env.local.example` ‚Äî documents required env vars

### Seed data (Hour 0.5‚Äì1)

- `seed/manufacturing.sql` ‚Äî table DDL + ~2000 rows of realistic data across the 5 tables. Mix of production runs (some failed, some succeeded), deviations with categories (temperature, pressure, contamination, alignment), equipment with maintenance histories, operators across shifts, quality checks tagged in/out of spec.
- `seed/run.ts` ‚Äî connects to `loom_demo_source`, runs the SQL

### Catalog (Hour 1‚Äì1.5)

- `src/lib/catalog/schema.ts` ‚Äî Drizzle schema:
  - `sources`, `tables`, `columns`, `column_profiles`, `docs` (markdown blobs + provenance), `embeddings` (vector(1536) ‚Äî optional populate), `audit_log`
- `src/lib/catalog/db.ts` ‚Äî Drizzle client
- `src/lib/catalog/push.ts` ‚Äî programmatic `db push` script
- `loom-catalog/.gitkeep` ‚Äî generated docs land here

### Worker ‚Äî Loop 1: Structural crawl (Hour 1.5‚Äì2.5)

- `src/lib/worker/sources/postgres.ts` ‚Äî connects to source DB, lists tables/columns/FKs from `information_schema`
- `src/lib/worker/profile.ts` ‚Äî per-column profiling (null rate, distinct count, min/max, top-5, histogram for numerics)
- `src/lib/worker/loop1.ts` ‚Äî orchestrates: discover tables ‚Üí profile each ‚Üí write `catalog.tables/columns/column_profiles` rows ‚Üí write structural markdown to `loom-catalog/`
- `src/lib/worker/markdown.ts` ‚Äî markdown writer with provenance tagging

### Worker ‚Äî Loop 2: Semantic enrichment (Hour 2.5‚Äì3.5)

- `src/lib/worker/loop2.ts` ‚Äî for each table: load structural doc + 5 sample rows + column profile ‚Üí call Claude Sonnet 4.6 ‚Üí parse response ‚Üí write `<!-- provenance: claude -->` paragraphs
- `src/lib/worker/anthropic.ts` ‚Äî raw Anthropic SDK client (NOT agent SDK), shared between Loop 2 and any other offline LLM calls
- `src/lib/worker/embeddings.ts` ‚Äî if `OPENAI_API_KEY` set, embed each doc and upsert into `catalog.embeddings`. Otherwise no-op.

### Agent (Hour 3.5‚Äì4.5)

- `src/lib/agent/tools/list_tables.ts`
- `src/lib/agent/tools/describe_table.ts`
- `src/lib/agent/tools/sample_rows.ts`
- `src/lib/agent/tools/run_sql.ts` ‚Äî uses `node-sql-parser` to assert single SELECT statement, no DML/DDL, then executes via pg
- `src/lib/agent/system-prompt.ts` ‚Äî builds system prompt with catalog index injected
- `src/lib/agent/loop.ts` ‚Äî Agent SDK loop, model `claude-opus-4-7`
- `src/app/api/chat/route.ts` ‚Äî POST, streams agent responses to client

### UI (Hour 4.5‚Äì5.5)

- `src/app/layout.tsx`, `src/app/page.tsx` ‚Äî single page, two-column layout
- `src/components/chat-panel.tsx` ‚Äî uses `useChat` from `ai/react`, suggested questions chips
- `src/components/catalog-panel.tsx` ‚Äî right side, lists tables with status (profiling/enriching/ready), polls `/api/catalog` every 1s
- `src/components/table-card.tsx` ‚Äî click expands, shows the markdown with provenance tags color-coded (schema=blue, query-log=purple, claude=green, human=orange)
- `src/app/api/catalog/route.ts` ‚Äî GET, returns current catalog state for the panel
- `src/app/api/profile/route.ts` ‚Äî POST, triggers Loop 1 manually (also auto-triggered on boot)
- `src/app/api/enrich/route.ts` ‚Äî POST, triggers Loop 2 for a given table

### Boot orchestration (Hour 5.5‚Äì6)

- `src/lib/boot.ts` ‚Äî on first request: ensure catalog schema, kick off Loop 1 for the demo source, kick off Loop 2 after Loop 1 completes
- Demo polish: copy review, suggested questions, color tweaks, screenshot for fallback

## Demo script (3 min)

1. "Loom is to tables what Claude Code is to files." (10s)
2. Boot the app ‚Üí catalog panel shows 5 tables flipping from `profiling` ‚Üí `enriching` ‚Üí `ready`. (30s)
3. Click on `deviations` ‚Üí markdown appears with color-coded provenance. (20s)
4. Ask: "What's our deviation rate by production line over the last 30 days?"
   ‚Üí agent calls `describe_table`, `run_sql`, returns a table + chart. (60s)
5. Ask: "Which equipment is most often involved in temperature deviations?"
   ‚Üí multi-table join answer, agent cites the doc it relied on. (45s)
6. "Everything you saw was generated in the last 5 minutes. No training,
   no manual catalog work. The system was always preparing." (15s)
