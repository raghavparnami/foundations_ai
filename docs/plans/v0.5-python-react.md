# v0.5 — Python FastAPI backend + React (Vite) frontend

**Status**: in progress (Vertical 1: Connections)
**Why**: Caller-controlled stack; tighter dependency surface; no Next.js
runtime; explicit accounting of every outbound network call.

## Hard rules

- **No regressions.** Old Next.js app on :3001 keeps running until parity
  is reached. New stack runs side-by-side. Both connect to the same
  `foundation_ai.*` schema in `loom_catalog` (Postgres on :5544). Cutover
  is changing where the React frontend points — nothing in the catalog
  needs to move.
- **External calls accounted explicitly.** Only two libs may make outbound
  HTTP: `httpx` (used by `pydantic-ai` for OpenRouter) and `python-gitlab`
  (for repos the user connects). Everything else is in-process. Anaconda /
  pip + npm telemetry disabled where possible.
- **No leftovers on disk.** All Loom-owned state stays in `foundation_ai.*`.
  No mirror files, no JSON caches, no local sqlite. Same rule the user
  set when we moved to `foundation_ai`.

## Tech choices

### Backend
- **fastapi 0.115+** + **uvicorn[standard]** — ASGI server, in-process workers.
- **psycopg 3** with the `pool` extra — async-capable; `search_path =
  foundation_ai, public` set on every checkout.
- **pydantic v2** — request/response models.
- **pydantic-settings** — env via `BaseSettings`.
- **pydantic-ai** — agent loop + tool calling. Provider configured for
  OpenRouter via the `OpenAIModel` with custom `base_url`.
- **httpx** — only via pydantic-ai's transport; explicit `Timeout(30.0)`.
- **sqlglot** — SQL parsing for the SELECT-only guard. Replaces
  node-sql-parser. Pure Python, no network.
- **pypdf** (PDF), **python-docx** (DOCX), **markdown-it-py** (MD). All pure
  Python.
- **python-gitlab** — GitLab REST. Only outbound to the user-configured
  GitLab host.
- **python-pptx** — PPTX generation. Local.
- **apscheduler 3.x** — in-process cron-like scheduler for Loop 1/2/4 +
  wiki agents.

### Frontend
- **React 18** (NOT 19 RC — we hit too many AI-SDK + RC issues). Switch
  later when the ecosystem stabilizes.
- **Vite 5** + **TypeScript strict**.
- **TanStack Router** — typed routes, file-based. Cleaner than react-router-dom.
- **TanStack Query** — server-state cache. Replaces ad-hoc useEffect+fetch.
- **Tailwind CSS 3** — same design tokens (`--accent` etc.) lifted into
  `frontend/src/styles/globals.css`.
- **react-markdown** + **remark-gfm** — same.
- **recharts** + **html-to-image** — same.
- **eventsource-parser** — for parsing SSE in the chat panel.

## Dev workflow

```bash
# Terminal 1: keep Postgres + old Next.js running (until parity)
docker compose up -d
npm run dev               # Next.js on :3001 (legacy)

# Terminal 2: new Python backend
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
uvicorn app.main:app --reload --port 8001

# Terminal 3: new React frontend
cd frontend
npm install
npm run dev               # Vite on :5173, points VITE_API_URL=http://localhost:8001
```

The legacy and new app coexist. Cutover is one env-var flip in the React
frontend's API base URL — no DB migration, no double-write, nothing
disruptive.

## Risk register

- **Streaming chat**: AI SDK's UIMessage stream protocol is non-trivial to
  reproduce. Pydantic-AI streams events; we'll translate them into the same
  SSE shape so existing frontend code can be lifted. If parity is too
  expensive, we can simplify to text-only streaming for the new frontend.
- **Tool calling parity**: AI SDK tools auto-validate inputs against the
  zod schema and pass them to `execute`. Pydantic-AI does the same with
  pydantic models. Behaviour-equivalent; syntax differs.
- **Scheduler concurrency**: Node uses `setInterval`. APScheduler runs in
  its own thread; we need to guard against the same tick running twice
  if uvicorn reloads. Use the AsyncIOScheduler + `pre_existing` lock.
- **CORS**: backend on :8001, frontend on :5173 — `FastAPI(CORSMiddleware)`
  with allowed origin `http://localhost:5173` for dev.
